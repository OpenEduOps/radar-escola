# Contribuindo

Obrigado por contribuir com o Radar Escola.

Este projeto aceita contribuicoes humanas, assistidas por automacao ou revisadas
com apoio de ferramentas, desde que sejam pequenas, compreensiveis e revisaveis.

## Metadados Publicos

Branches, titulos de pull request, mensagens de commit e mensagens de merge
devem descrever o produto, a issue, o comportamento ou a mudanca de engenharia.

Nao use nomes de agentes ou ferramentas de programacao nesses metadados
publicos. A politica nao tenta provar se uma contribuicao foi criada com ajuda
de automacao. Isso nao e tecnicamente confiavel e nao e o objetivo.

O objetivo e manter o historico publico focado no produto.

Exemplos de metadados bons:

- `metadata-policy-guard`
- `Add public metadata policy`
- `Improve setup documentation`
- `Validate first-use requirements`

Mantenedores nao devem fazer merge de pull requests que falhem na guarda de
metadados publicos.

## Antes De Abrir PR

- Use branch com nome neutro e focado no produto.
- Use titulo de PR focado na mudanca.
- Use mensagens de commit focadas no comportamento ou documentacao.
- Rode os checks locais possiveis.
- Atualize documentacao quando o comportamento mudar.
